Domain-Driven Design (DDD)

Lenguaje Ubicuo

Cliente (Customer)
Entidad raíz que realiza pedidos. Identificado por CustomerId.
Producto (Product)
Entidad con precio (Money) y stock disponible.
Pedido (Order)
Agregado raíz. Contiene líneas de pedido (OrderLine) y gestiona su ciclo de vida: pendiente → confirmado → enviado → entregado.
Dinero (Money)
Value Object inmutable con validación de moneda.
Email
Value Object con validación de formato.
Dirección (Address)
Value Object inmutable con componentes estructurados.
